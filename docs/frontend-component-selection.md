# 前端组件库选型

## 结论

前端基础组件库选用 Ant Design Vue。未来可视化能力按模块引入 AntV 和 Apache ECharts，不把脚本可视化编辑、监控图表、报告预览和结果对比压到单一 UI 组件库上。

## 选型理由

本平台后续重点是脚本可视化编辑、监控视图、堆栈异常分析、报告可视化预览编辑和多次性能测试结果对比。普通后台组件库只能覆盖表单、表格、弹窗、布局和反馈组件，无法单独支撑复杂图编辑、时序监控、多维对比和报告编排。

Ant Design Vue 作为基础 UI 层，负责后台工作台、表单、表格、抽屉、弹窗、标签页、分段控制、空状态和消息反馈。AntV 和 Apache ECharts 作为可视化底座，按业务模块逐步落地。

## 模块对应

| 未来模块 | 前端底座 |
|----------|----------|
| 脚本可视化编辑 | Ant Design Vue + AntV X6 |
| 监控视图 | Ant Design Vue + Apache ECharts |
| 堆栈异常分析 | Ant Design Vue + AntV G6 |
| 报告可视化预览编辑 | Ant Design Vue + AntV G2 + Apache ECharts |
| 多次性能测试结果对比 | Ant Design Vue + Apache ECharts + AntV S2 |

## 落地规则

1. 现有已开发前端模块迁移到 Ant Design Vue，避免长期混用多套基础 UI 组件库。
2. AntV 和 ECharts 只在对应可视化模块真正落地时引入，不提前堆入运行包。
3. 基础 UI 不自研表格、表单、弹窗、抽屉、分页、消息确认等通用能力。
4. 可视化编辑、监控图表、报告预览和结果对比优先复用成熟可视化库，只在业务交互层做定制。
5. 样式层保留项目自己的页面布局和业务视觉规范，组件库只作为交互控件底座。
