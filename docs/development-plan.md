# 性能测试平台 - 阶段性开发计划

## 当前策略

平台按长期可扩展方案推进，但每个阶段都必须交付可运行、可验证的闭环能力。第一阶段聚焦平台骨架、登录、项目管理、项目成员和基础权限，为脚本、执行、报告模块提供统一归属。

## Phase 1：平台骨架与项目资产

目标：完成内部使用所需的基础工作台。

范围：

1. Spring Boot 3 + Vue 3 + Element Plus 单体骨架。
2. 演示登录和用户持久化。
3. 项目创建、查询、归档、恢复。
4. 项目成员查看、添加和负责人权限约束。
5. H2 文件库持久化，开发期自动建表。

验收：

1. 用户可登录平台。
2. 用户可创建项目，重启后项目仍存在。
3. 项目负责人可添加成员。
4. 非负责人不能执行归档、恢复等负责人操作。
5. 前端页面可完成上述操作。

## Phase 2：脚本管理与单机执行

目标：跑通 JMeter JMX 上传到单机执行的闭环。

范围：

1. JMX 文件上传和文件安全校验。
2. 脚本版本记录。
3. 基础执行配置。
4. 单机执行队列。
5. 执行状态查询和取消。
6. 执行日志与失败原因记录。

验收：

1. 用户可在项目下上传 JMX。
2. 用户可选择脚本版本并提交执行。
3. 系统一次只运行一个本机 JMeter 任务。
4. 页面刷新后仍可查询任务状态。

## Phase 3：报告与结果可视化

目标：形成可交付的基础压测报告。

范围：

1. JMeter 结果文件解析。
2. TPS、响应时间、错误率、采样统计。
3. 基础图表。
4. Markdown 和 HTML 报告生成。
5. 报告下载和项目内查看。

验收：

1. 每次完成的执行记录可生成报告。
2. 报告可下载。
3. 报告能关联项目、脚本版本、执行配置和执行记录。

## 后续增强

1. 监控采集和报告整合。
2. 造数工厂。
3. 函数库。
4. Git 关联。
5. AI 分析。
6. 分布式执行。
