# 报告管理模块

## 1. 模块定位

报告管理负责把执行结果解析为可交付的测试报告，并提供项目内查看、下载和追溯能力。它不负责启动压测，也不直接修改执行记录状态。

## 2. 职责边界

| 负责 | 不负责 |
|------|--------|
| JMeter 结果文件解析 | JMeter 执行和停止 |
| 聚合指标计算 | 原始脚本版本管理 |
| Markdown/HTML 报告生成 | 监控 Agent 采集 |
| 报告权限和下载 | AI 自动分析 |

## 3. 需求规格

### REQ-REPORT-001 生成基础测试报告

**用户故事**：作为性能测试工程师，我希望任务结束后自动生成报告，用于交付测试结果。

**功能说明**：

1. 任务成功、失败或已取消后，平台可基于结果文件生成报告。
2. 报告包含任务信息、执行配置、聚合指标、接口明细、错误摘要和结论备注。
3. 聚合指标至少包括 TPS、平均响应时间、P90、P95、P99、最大响应时间、最小响应时间、错误率、采样总数。
4. MVP 支持 Markdown 和 HTML 两种报告查看或下载方式。

**输入/输出**：

| 类型 | 内容 |
|------|------|
| 输入 | 执行记录 ID、报告标题、结论备注 |
| 输出 | 测试报告、报告文件、报告链接 |

**异常场景**：

1. 结果文件不存在时报告生成失败并提示原因。
2. 结果文件过大时异步生成报告，并展示生成状态。
3. 报告生成失败时允许重新生成。

**验收标准**：

1. 已结束任务可以生成一份基础测试报告。
2. 报告中能看到 TPS、响应时间分位值、错误率和采样总数。
3. 用户可以下载 Markdown 或 HTML 报告文件。

### REQ-REPORT-002 报告分享和追溯

**用户故事**：作为测试负责人，我希望报告可以被项目成员访问，并能追溯到原始任务和脚本版本。

**功能说明**：

1. 报告与项目、任务、执行记录、脚本版本关联。
2. 项目成员可查看项目内报告。
3. 报告页面提供原始任务入口和脚本版本信息。

**输入/输出**：

| 类型 | 内容 |
|------|------|
| 输入 | 报告 ID、访问用户 |
| 输出 | 报告详情、权限校验结果 |

**异常场景**：

1. 非项目成员访问报告时返回无权限。
2. 原始任务被归档时报告仍可查看。

**验收标准**：

1. 项目成员可以通过报告链接查看报告。
2. 报告详情能跳转回对应任务详情。
3. 非项目成员无法查看报告内容。

## 4. 关键实体

| 实体 | 关键字段 | 说明 |
|------|----------|------|
| `test_report` | `id`, `execution_id`, `title`, `format`, `file_path`, `status`, `summary`, `created_by`, `created_at` | 测试报告 |
| `report_metric` | `id`, `report_id`, `scope`, `label`, `sample_count`, `error_rate`, `avg_rt`, `p90`, `p95`, `p99`, `tps` | 报告指标 |

报告状态：

```text
GENERATING -> SUCCESS
GENERATING -> FAILED
SUCCESS -> REGENERATING -> SUCCESS
```

## 5. 接口草案

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/executions/{executionId}/reports` | 生成报告 |
| `GET` | `/api/projects/{projectId}/reports` | 项目报告列表 |
| `GET` | `/api/reports/{reportId}` | 报告详情 |
| `GET` | `/api/reports/{reportId}/download?format=markdown` | 下载 Markdown |
| `GET` | `/api/reports/{reportId}/download?format=html` | 下载 HTML |
| `POST` | `/api/reports/{reportId}/regenerate` | 重新生成报告 |

## 6. 详细设计调整点

1. 报告生成只读取 `execution_result` 和执行配置快照，不回读可变脚本参数。
2. 原始结果解析要支持大文件流式处理，避免一次性加载完整 JTL。
3. 报告文件存储到 `storage/reports/{projectId}/{executionId}/{reportId}`，和原始结果文件分离。
4. 报告详情展示以数据库指标为主，文件下载以生成文件为主。
5. 报告链接访问仍需要项目成员权限，MVP 不提供匿名公开分享。
