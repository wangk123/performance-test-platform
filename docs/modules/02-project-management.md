# 项目管理模块

## 1. 模块定位

项目管理是平台资产归属入口。脚本、执行任务、执行结果、报告、监控目标、造数模板和函数资产都必须挂靠到项目或系统公共库。

## 2. 职责边界

| 负责 | 不负责 |
|------|--------|
| 项目创建、编辑、归档、恢复 | 用户账号生命周期 |
| 项目成员管理 | 企业组织同步 |
| 项目负责人权限约束 | 脚本文件解析 |
| 项目资产访问入口 | 任务调度和报告生成 |

## 3. 需求规格

### REQ-PROJ-001 创建和维护项目

**用户故事**：作为测试负责人，我希望为每个被测系统创建独立项目，以便集中管理脚本、任务和报告。

**功能说明**：

1. 支持创建、编辑、查询、归档和恢复项目。
2. 项目字段包括项目名称、项目编码、描述、负责人、状态、创建时间、更新时间。
3. 项目编码在系统内唯一。
4. 归档项目默认不出现在新建任务选择列表中。

**输入/输出**：

| 类型 | 内容 |
|------|------|
| 输入 | 项目名称、项目编码、描述、负责人 |
| 输出 | 项目详情、项目列表、操作结果 |

**异常场景**：

1. 项目编码重复时拒绝保存。
2. 项目名称为空时拒绝保存。
3. 已有关联任务的项目只允许归档，不允许物理删除。

**验收标准**：

1. 用户可以创建一个项目，并在项目列表中看到该项目。
2. 重复项目编码无法保存，并显示明确错误信息。
3. 归档项目后，该项目不再出现在新建测试任务的项目选择项中。

### REQ-PROJ-002 成员与基础权限

**用户故事**：作为测试负责人，我希望控制项目成员范围，避免无关用户修改测试资产。

**功能说明**：

1. 支持为项目添加和移除成员。
2. MVP 提供三类项目访问身份：管理员、项目负责人、项目成员。
3. 管理员可管理所有项目。
4. 项目负责人可管理自己负责的项目。
5. 项目成员可查看项目并提交测试任务。

**输入/输出**：

| 类型 | 内容 |
|------|------|
| 输入 | 项目 ID、用户 ID、项目角色 |
| 输出 | 成员列表、权限校验结果 |

**异常场景**：

1. 移除项目负责人前必须先指定新的负责人。
2. 非项目成员访问项目详情时返回无权限。
3. 已归档项目不允许新增成员，除非先恢复项目。

**验收标准**：

1. 项目负责人可以添加项目成员。
2. 非项目成员无法查看该项目下的脚本、任务和报告。
3. 管理员可以查看和维护所有项目。

## 4. 关键实体

| 实体 | 关键字段 | 说明 |
|------|----------|------|
| `project` | `id`, `code`, `name`, `owner_id`, `status`, `description`, `created_by`, `created_at`, `updated_by`, `updated_at` | 项目主表 |
| `project_member` | `id`, `project_id`, `user_id`, `project_role`, `created_by`, `created_at` | 项目成员 |

状态：

```text
ACTIVE -> ARCHIVED
ARCHIVED -> ACTIVE
```

项目不做物理删除，避免脚本、任务、报告失去归属。

## 5. 接口草案

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/projects` | 项目列表，可选择是否包含归档项目 |
| `POST` | `/api/projects` | 创建项目 |
| `GET` | `/api/projects/{projectId}` | 项目详情 |
| `PUT` | `/api/projects/{projectId}` | 编辑项目 |
| `PATCH` | `/api/projects/{projectId}/archive` | 归档项目 |
| `PATCH` | `/api/projects/{projectId}/restore` | 恢复项目 |
| `GET` | `/api/projects/{projectId}/members` | 成员列表 |
| `POST` | `/api/projects/{projectId}/members` | 添加成员 |
| `DELETE` | `/api/projects/{projectId}/members/{userId}` | 移除成员 |

## 6. 详细设计调整点

1. 项目编码作为业务唯一键，后续报告链接、目录结构和审计日志均可引用项目编码。
2. 项目负责人既是 `project.owner_id`，也应在 `project_member` 中拥有 `OWNER` 角色，便于统一查询。
3. 项目归档只影响新增业务入口，不影响历史脚本、任务、报告查看。
4. 项目成员权限作为所有业务模块的前置校验，脚本、执行和报告模块不得重复维护成员关系。
5. 后续如接入组织架构，只替换成员来源，不改变项目资产归属模型。
