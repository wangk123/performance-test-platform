## Context

造数工厂目前仅有文档草案与 Mock API，项目页为占位。函数库已提供部分格式化随机能力（如手机号、身份证），但只服务压测运行时表达式，不能解决「测环境批量前置写库」。

本设计落地探索结论：语义核心（字段角色 / 关联 / 格式）+ 可插拔采集；V1 用表快照 Diff；产出仅写库；确认硬门禁；细粒度权限后置。

## Goals / Non-Goals

**Goals:**

- 用户配置测环境 MySQL 数据源与库表过滤后，通过多样本录制得到库变更模板。
- 系统推断字段角色与关联，并以 HIGH/MEDIUM/LOW + 依据展示；用户确认后方可克隆。
- 按已确认模板批量 INSERT/UPDATE 写库，支持失败策略与任务审计。
- 采集接口可替换（预留 binlog），V1 实现 SNAPSHOT。

**Non-Goals:**

- DELETE 克隆、binlog 采集实现、CSV/JSON 导出、业务 API 编排灌数。
- 细粒度 RBAC、生产库写入、与压测任务自动绑定。
- 无主键大表的自动可靠克隆（可进确认页标风险，默认不自动执行）。

## Decisions

### D1: 主路径为「录制 → 推断 → 确认 → 写库」，而非手写规则或 API 回放

- **选择**：业务点一次 + 库变更录制；手写规则仅作确认页修补；API 灌数不做。
- **原因**：降低建模成本；避免长链路/限流阻塞造数；测环境允许直连写库。
- **备选**：纯字段规则引擎（成本高）；HTTP 录制回放（再陷 API 瓶颈）。

### D2: 采集可插拔；V1 = 快照 Diff

- **选择**：`CaptureProvider = SNAPSHOT | BINLOG`；V1 只实现 SNAPSHOT。
- **原因**：快照无 binlog 依赖；语义层与管道解耦，后期可换 CDC。
- **备选**：binlog 优先（环境要求高，推迟）。

### D3: 过滤策略 = 精确 + 表达式，exclude 优先，且必须至少一条 include

- **选择**：`include`/`exclude` 支持精确 `db.table` 与表达式（通配/正则）；求值后得到采集表集 S；禁止空 include 扫全库。
- **原因**：控制噪声与大表成本，降低误录风险。

### D4: 置信为离散档位 + 规则依据，非数值模型分

- **选择**：HIGH / MEDIUM / LOW，每条附可展示依据（元数据 / 多样本 / 跨表同值 / 形态）。
- **原因**：可解释、可测；避免假精度。
- **门禁**：LOW 须人工改角色或显式采纳；UNIQUE_REGEN / FORMATTED_RAND 必须绑定生成器方可确认。

### D5: 字段角色与操作模型

- 角色：`LITERAL`、`UNIQUE_REGEN`、`FK_REF`、`BIZ_KEY`、`FORMATTED_RAND`、`TIMESTAMP`、`UPDATE_KEY`、`UPDATE_SET`、`IGNORE`。
- 操作：仅 `INSERT`、`UPDATE`；不做 `DELETE`。
- 克隆时维护批内 `id_map`，供 FK_REF / BIZ_KEY 重写。
- 生成器优先绑定既有函数库能力（同名语义），在平台侧 Java 实现写库生成，不依赖 JMeter 进程。

### D6: 模板确认后版本只读

- **选择**：确认生成 `template_version`；再录制或大改产生新版本；CloneJob 绑定版本。
- **原因**：避免执行中途模板漂移。

### D7: 克隆执行

- 按依赖排序写库；默认每克隆份（batch）一事务；单批失败默认继续下一批（可配置遇错即停）。
- 限制单任务最大克隆份数与时长。
- V1 权限：仅需已登录且能访问项目；细粒度角色后置。

### D8: 模块与 API 归属

- 后端独立 package（如 `seed` / `datafactory`），项目下 REST：
  - 数据源 CRUD、过滤策略
  - 录制会话（开始/样本结束/结束/触发推断）
  - 模板草稿/确认
  - 克隆任务创建/查询
- 替换 `ModuleMockController` 中造数相关 Mock。
- 前端替换 `ProjectDetail` 造数占位。

## Risks / Trade-offs

- [快照噪声 / 后台写] → 强制过滤 + 多样本共识 + 确认页；表级共识阈值可配。
- [无正式 FK 的业务键误判] → MEDIUM/LOW + 强制确认；允许用户改关联。
- [大表快照成本] → include 必填；无主键表默认不自动克隆。
- [唯一键冲突 / 格式被业务校验拒绝] → 确认前强制生成器；FORMATTED_RAND 绑格式生成器。
- [UPDATE 定位错误] → UPDATE_KEY 必须确认；低置信阻断。
- [凭证泄露] → 加密存储 + 审计；文档强调仅测环境（实现上可后续加 host 黑名单）。

## Migration Plan

1. 新增表与 API，前端 Tab 切换到新 UI。
2. 移除或停用造数 Mock 接口，避免双轨。
3. 回滚：停用新 API/UI，恢复占位与 Mock（无历史生产数据依赖）。

## Open Questions

- 表达式语法最终采用 glob 还是正则（或二者都支持）——实现时在过滤模块定一种默认并在 API 文档写明。
- 克隆默认最大 N 的具体数值——实现时给合理默认（如 10000）并做成可配置。
