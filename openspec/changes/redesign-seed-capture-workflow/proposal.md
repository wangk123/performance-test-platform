## Why

当前录制将配置、执行和结果合并为一次同步会话：保存配置即开始扫描，长时间录制没有进度，结束后也没有可查询的样本记录与分页详情。为了支持大体量存量数据库的多次样本采集和可靠模板推断，需要将录制重构为可复用策略、异步快照样本和手动多样本分析。

## What Changes

- 新增可重复执行的录制策略；保存策略不触发采集，用户手动执行后才创建独立样本。
- 将全库快照改为异步长任务，同一数据源只允许一个活跃录制，单次录制内部支持线程数和每批行数配置、按表及可用主键范围并行。
- 新增录制状态、表级进度、取消、中断识别、磁盘低水位保护，以及失败/取消样本的部分数据保留。
- 新增永久样本记录与分页详情，按采集开始时间和策略内序号稳定排序，并支持手动删除。
- 快照大数据写入本地持久化磁盘的压缩分片文件；平台数据库保存策略、状态、进度、库表元数据、分片清单和逻辑指纹。
- 新增手动多样本分析：至少选择同一策略的三份终态样本，按采集时间排序后对相邻快照执行 Diff。
- Diff 使用表级指纹粗筛、变化分片定位和行级细筛；无主键或不完整数据按风险/UNKNOWN 处理，不静默推断。
- 新增异步分析进度、取消、永久分析记录和分页结果；分析完成后样本、分析结果和模板相互独立，不级联删除。
- **BREAKING**：移除“创建录制会话即同步开始并在结束时自动推断”的现有录制流程与 API 语义，旧 `seed_capture_session` 数据不迁移并在升级时清理。

## Capabilities

### New Capabilities

- （无）

### Modified Capabilities

- `seed-capture`: 将即时录制会话改为“可复用策略 → 异步快照样本 → 手动相邻样本 Diff → 模板草稿”的可追踪工作流，并增加大数据分片、进度、分页、取消、恢复和清理约束。

## Impact

- 后端：重构 `seed` 录制服务、任务调度、状态机、样本存储、Diff 分析和清理流程；增加录制与分析查询 API。
- 数据库：新增策略、样本、表/分片清单、任务心跳、分析记录等持久化结构；清理旧录制会话数据。
- 文件系统：使用现有持久化存储目录保存样本与分析结果分片，需要容量统计、低水位保护、校验与原子写入。
- 前端：重做造数工厂“录制”页，增加策略管理、执行进度、样本历史、分页详情、样本选择和分析记录。
- 模板：模板仍由分析结果生成草稿，既有确认门禁和克隆流程保持不变。
- 依赖：优先复用现有 JSON 能力和 JDK GZIP，不引入新的外部存储服务。
